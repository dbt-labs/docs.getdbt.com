The seeds object allows you to query information about a all seeds in a given job.

### Arguments

When querying for `seeds`, the following arguments are available. Note that if you do not include a runId, it will default to the most recent run of the specified job:

<ArgsTable queryName="seed" />

Below we show some illustrative example queries and outline the schema of this seeds object.

### Example Queries
#### Seeds information

The query below pulls relevant information about all seeds in a given job.  For example, we could see the load times.

```graphql
{
  seeds(jobId: 123) {
    uniqueId
    name
    executionTime
    status
  }
}
```
