The tests object allows you to query information about all tests in a given job.


### Arguments

When querying for `tests`, the following arguments are available. Note that if you do not include a runId, it will default to the most recent run of the specified job:

<ArgsTable queryName="tests" />

Below we show some illustrative example queries and outline the schema (all possible fields you can query) of this tests object.

### Example Queries
#### Tests result

The example query below finds all tests in this job, and includes information about those tests.

```graphql
{
  tests(jobId: 123) {
    runId
    accountId
    projectId
    uniqueId
    name
    columnName
    state
  }
}
```
